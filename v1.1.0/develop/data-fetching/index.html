Data Fetching

This is experimental and only works for local development. Data fetching in production environments will be supported in early 2020.

In the course of developing data experiences on the Looker platform, you will find it useful to fetch production data in the lens development playground. To facilitate that, we've set up a local proxy server which will forward api requests to any target looker instance you have permissions to view.

Proxy Configuration

Instructions for configuring and starting the lens proxy can be found on our packages/Server README.

To validate that the proxy server is configured correctly it's helpful to query the authenticated user, which can be tested with the following example:

;() => {
  const [user, setUser] = useState()
  function fetchUser() {
    fetch('/api/user')
      .then((response) => response.json())
      .then((state) => setUser(state))
  }
  return (
    <>
      <Button onClick={fetchUser}>Fetch User Data</Button>
      {user && (
        <UnorderedList>
          <li>
            <strong>ID:</strong> {user.id}
          </li>
          <li>
            <strong>Name:</strong> {user.display_name}
          </li>
          <li>
            <strong>Email:</strong> {user.email}
          </li>
          <li>
            <strong>Home Space:</strong> {user.home_space_id}
          </li>
          <li>
            <strong>Role IDs:</strong> {JSON.stringify(user.role_ids)}
          </li>
        </UnorderedList>
      )}
    </>
  )
}
